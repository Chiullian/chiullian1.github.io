Water

描述

有 N 个容量无穷大的水壶，它们从 $1\sim N$ 编号，初始时 i 号水壶中装有 A_i单位的水。

你可以进行不超过 $K$ 次操作，每次操作需要选择一个满足 $1 \leq x \leq N-1$ 的编号 $x$，然后把 $x$ 号水壶中的水全部倒入 $x + 1$ 号水壶中。

最后你可以任意选择恰好一个水壶，并喝掉水壶中所有的水。现在请你求出，你最多能喝到多少单位的水。

取走k数

描述

小明有一个大小为$n$的数组,给定$k$ 他执行下列操作$m$次

找到当前数组中第$k$小的数, 拿走这个数(之后的操作中就没有这个数了), 请你求出最终所有拿出的数之和

比如$n=6$,数组为$3,3,2,2,1,1$, $k$为2, $m$为3, 则第一次取出$1$, 此时数组为$3,3,2,2,1$, 第二次取出$2$, 此时数组为$3,3,2,1$, 第三次取出$2$, 最终和为1+2+2=5

排个序取第k个往后m个数的和即可

Large

Lzh大师是树上 Dynamic Programming 的高手，他给小 s 出了道简单的树上问题。

一个有根树，有 N 个节点，根的节点编号是 1 。

每一个节点i 都有一个数值 $A_i$ 。

每个节点只能向以他为根的子树里的节点跳跃。

从 i 号节点跳到j 号节点的代价是$2^{ dep[j] - dep[i] - 1} * A_j$* 。

每次跳跃的代价相加。

问从根节点到叶子节点的最小代价是多少。 （任何叶子节点都是可以的）。

输入

三行输入。

第一行一个整数 $N$，表示有 N 个节点构成一棵树。

第二行 $N - 1$个整数，第 $i$个整数 $p_i$ 表示第 $i + 1$号节点的父亲是$p_i$。

第三行 $N$个整数，第 i 个整数 $A_i$ 表示第 i 号节点的数值是$A_i$ 。

$1 \leq N \leq$

$1 \leq p_i \leq N$

$0 \leq A_i \leq 1000$

输出

一行一个整数，表示从根节点到叶子节点的最小代价。

输出样例 1

树上dp

从根节点往上跳的

如果 $A[i]$不是 0 ，那么 号节点只能向下跳有限次。 往下多跳一格, 代价就乘2, 发现代价增长极快,而${A_i}$最大为1000, 那么我们在每一个点只 需要枚举跳几次即可, 特别注意当 为0时, 无论从哪个点跳过来, 代价都是0.

极端情况, 都在一条链上最多跳不能超过 64次 $2 ^{64}$ 太大了而且$A_{i}$才1000所以设置一个限制往上跳最多30+层就不跳了
